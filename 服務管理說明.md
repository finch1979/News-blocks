# 方塊磚新聞 - 常駐服務設定指南

## 📋 功能說明

此應用程式提供多種啟動方式，可以讓服務在背景持續運行，不受 VSCode 或命令提示字元視窗關閉影響。

## 🚀 快速開始

### 方法一：手動啟動背景服務（推薦）

1. **啟動服務**
   - 雙擊執行 `start.bat`
   - 服務將在背景運行
   - 瀏覽器會自動開啟 http://127.0.0.1:7789
   - ✅ 可以安全關閉命令提示字元視窗
   - ✅ 若服務已在執行，腳本只會直接開啟網頁，不會重複開第二份

### 方法一之一：一鍵開啟 APP

1. **直接開啟服務與網站**
   - 雙擊執行 `open_app.bat`
   - 如果服務尚未啟動，會先在背景啟動
   - 接著自動打開 http://127.0.0.1:7789
   - 適合當作平常的桌面啟動器

2. **停止服務**
   - 雙擊執行 `stop.bat`
   - 所有相關進程將被終止

### 方法二：設定開機自動啟動

1. **安裝開機啟動**
   - 雙擊執行 `install_autostart.bat`
   - 服務將在每次登入 Windows 時自動在背景啟動
   - 無需任何視窗，完全靜默運行
   - 會在桌面建立 `News Blocks.url` 網址捷徑，可直接點開

2. **取消開機啟動**
   - 雙擊執行 `uninstall_autostart.bat`
   - 移除開機啟動設定

## 📊 服務管理

### 檢查服務狀態
```powershell
netstat -ano | findstr ":7789"
```
如果有輸出，表示服務正在運行

### 訪問網頁
- 本機訪問：http://127.0.0.1:7789
- 局域網訪問：http://<你的IP>:7789

### 查看日誌
如需查看服務日誌，可以臨時以前台模式運行：
```cmd
.venv\Scripts\python app.py
```

## 🔧 故障排除

### 問題：服務啟動失敗
**解決方案：**
1. 檢查 Python 是否已安裝：`python --version`
2. 檢查端口 7789 是否被占用：`netstat -ano | findstr ":7789"`
3. 如果被占用，執行 `stop.bat` 清理進程

### 問題：開機啟動無效
**解決方案：**
1. 確認 Windows 啟動資料夾中存在 `News Blocks.lnk`
2. 重新執行 `install_autostart.bat`
3. 開機後等待 10-20 秒讓服務完全啟動

### 問題：無法訪問網頁
**解決方案：**
1. 確認服務正在運行（見上方「檢查服務狀態」）
2. 清除瀏覽器快取並重新整理（Ctrl+F5）
3. 嘗試重啟服務：先執行 `stop.bat`，再執行 `start.bat`

## 📁 檔案說明

| 檔案 | 用途 |
|------|------|
| `start.bat` | 啟動背景服務 |
| `stop.bat` | 停止服務 |
| `install_autostart.bat` | 設定開機自動啟動 |
| `uninstall_autostart.bat` | 取消開機自動啟動 |
| `start_silent.vbs` | 靜默啟動腳本（由安裝程序自動建立） |
| `app.py` | 主程式 |
| `data/blocks.json` | 本機方塊資料儲存（不建議提交） |
| `data/blocks.example.json` | 公開範例資料 |

## ⚙️ 進階設定

### 修改端口號
如需修改端口（預設 7789），編輯以下檔案：
- `app.py`：最後一行的 `port=7789`
- `smoke_test.py`：`BASE` 變數
- `start.bat`：顯示的網址

### 允許局域網訪問
編輯 `app.py` 最後一行：
```python
uvicorn.run("app:app", host="0.0.0.0", port=7789, reload=False)
```
⚠️ 注意：開放給局域網後，請注意網路安全

## 💡 使用建議

2. **測試開發**：使用手動啟動模式，方便調試
3. **停用期間**：執行 `stop.bat` 停止服務節省資源
